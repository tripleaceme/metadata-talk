The Core Idea

Pipelines move data.

Metadata makes it usable.

What most companies overlook

Your Data Stack Has 3 Layers

1 Infrastructure — warehouses, pipelines, orchestration
2 Transformation — models, dbt, SQL
3 Metadata — the missing layer
➤ Most teams only optimize the first two
METADATA, really ?

What is Metadata?

Metadata is not just "extra info", it includes:

📄
Documentation
What does this mean?
🔘
Lineage
Where did it come from?
👤
Ownership
Who is responsible?
📈
Usage
Who depends on it?
★ Metadata = Context. It is the interface between humans and data.
Impact

Why It Matters

Without metadata:

  • Data is hard to trust
  • Data is hard to find
  • Data is hard to debug
➤ Data exists, but it's not usable
Breakdown

Where Things Break

✕ Documentation breaks — outdated, inconsistent
✕ Lineage is incomplete — no column-level visibility
✕ Discovery is difficult — tribal knowledge
✕ Usage is invisible — no feedback loop
Problem #1

Documentation is Broken

  • No tracking → becomes outdated the moment it's written
  • Rewritten everywhere → inconsistent across models
  • Nobody trusts it → so nobody reads it
🔍 Documentation without tracking is just hopeful notes
Solution Path

What Good Documentation Looks Like

Documentation should be a system, not a chore:

  • Track changes over time — know when definitions drift
  • Stay consistent across models — single source of truth
  • Be reusable, not duplicated — write once, inherit everywhere
➤ Not just static text — a living, tracked documentation layer

Finding Data Is Still Hard

💭 If your team Slacks each other to find data, your discovery layer is missing
Your Unique Angle

Usage Blindness

We document data... but we don't know:

? Who reads it — is anyone consuming this documentation?
? What is useful — which docs actually help people?
? What is ignored — effort wasted on unread descriptions
💥 No feedback loop. This is the blind spot most teams don't even know they have.

Solution #1

Better Documentation Systems

  • Track changes — know when descriptions were last updated and by whom
  • Maintain consistency — inherit definitions so they don't diverge
  • Reuse definitions across models — write once, apply everywhere
🛠 Tools like dbt Doc Tracker make this possible — tracking documentation changes, flagging staleness, and keeping your docs honest across your entire dbt project

Solution #3

Make Data Easy to Find

  • Context attached to datasets — descriptions, owners, freshness
  • Clear definitions — no more guessing what col_1 means
  • Reduced duplication — one canonical source, not five variations
➤ Good discovery is a byproduct of good documentation and lineage working together
Advanced Insight

Close the Loop

  • Track documentation consumption — who reads what?
  • Improve what people actually use — double down on value
  • Remove what doesn't matter — reduce noise
💡 This is where metadata evolves — from static context into an adaptive layer that improves based on real usage signals
